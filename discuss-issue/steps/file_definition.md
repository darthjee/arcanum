# Issue File Definition

## Extract ID and fetch content

Read [extract_id_and_name.md](extract_id_and_name.md) and follow the instructions there. The script resolves `ID`, `TITLE`, and `FILE` and guarantees `FILE` exists with content — there is no filename logic for the agent to apply manually. The id is always numeric and tied to a real GitHub issue; there is no local-only id convention.
